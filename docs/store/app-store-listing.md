- Do NOT repeat words already in the App Name or Subtitle (Apple ignores duplicates).

- High-volume primary targets: travel journal, trip tracker, world map, travel diary
- Medium-volume / high-intent: budget travel, countries visited, trip log

## A/B Test Plan (App Store Product Page Optimization)

Apple allows up to 3 simultaneous treatment tests against the default listing.

### Test 1: App Icon (launch T+7)
- **Control:** Map/globe icon with coral accent pin
- **Treatment A:** Pin cluster on blue world map background
- **Treatment B:** Minimal "TC" monogram on deep blue
- **Success metric:** Conversion rate (impressions to product page)
- **Duration:** 21 days minimum for statistical significance

### Test 2: First Screenshot (T+30)
- **Control:** World map (hero, current Screenshot 1)
- **Treatment:** Budget breakdown detail view (Screenshot 2 as hero)
- **Hypothesis:** Budget tracking may convert better for the search query
  "travel budget app" — a high-intent segment
- **Success metric:** Conversion rate segmented by search keyword

### Test 3: Subtitle (T+60)
- **Control:** "Map, Budget & Travel Memories"
- **Treatment A:** "Track Trips, Costs & Memories"
- **Treatment B:** "Your Personal World Travel Map"
- **Success metric:** Impressions (keyword discovery) + conversion rate combined

## Review Strategy

**When to request a review (in-app prompt trigger):**
- After user adds their 3rd trip (high-engagement signal, investment made)
- NOT on first launch
- NOT immediately after onboarding
- NOT after a negative event (crash, error)

**Prompt copy:**
> "You have added 3 trips. Enjoying Travel Checker so far?"
> [Yes, loving it] [Not yet]
>
> If "Yes": Native iOS review prompt (SKStoreReviewRequest)
> If "Not yet": "What can we improve?" → In-app feedback form → email to support@travelchecker.app

**Target:** Maintain 4.3+ average rating. Respond to every review within 48 hours.
Negative reviews should receive an empathetic reply + resolution path within 24 hours.
